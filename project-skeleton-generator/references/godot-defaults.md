# Godot 4 默认约定

## 何时使用

- 在空白工作区中生成全新骨架，并且需求里没有更明确的引擎说明时使用这一套默认约定。
- 在已有 Godot 项目中补结构时，优先贴合现有目录；只有现有项目没有约定时才回退到这里。

###（一）项目结构
- Core 文件夹：核心机制层
  - ex：量化实现的
  - global 文件夹：全局脚本
    - data 脚本：加载量化数据
    - sys 脚本: 场景实例化，对象实例化，加载游戏
  - script 文件夹: 一些工具脚本
    - weightConfig 脚本：权重配置资源
    - randomUtil 脚本：提供基于权重的随机选择方法
    - 核心对象.gd 类脚本
    - 核心对象.tscn 场景
    - mian.tscn：开始界面
    - main.gd：开始界面脚本
    - game.tscn: 选择界面
    - game.gd: 选择界面脚本
    - ......  
- Ex 文件夹： 量化实现层
  - 核心对象量化文件夹 例如：
  - Relics:
    - Common
      - 量化实例图片
      - 量化实例脚本
    - Uncommon
    - Rare
    - Legendary
  - Symbols
    - Common
      - 量化实例1 
      	- 量化实例1 图片
      	- 量化实例1 脚本
    - Uncommon
    - Rare
    - Legendary
- Res 资源层： 美术素材，动画，音乐音效
  - Picture
  - Music
  - Anim
- UI 层： 场景UI层，一些逻辑比较简单的场景及脚本
  - 图鉴.tscn
  - 图鉴.gd
  - 
###（二）tscn场景结构
- 目的：更了更好的替换手绘素材，方便调整
- 图片采用 textureRect占位
- 只有对于数组排列的对象才用 容器节点
- 一开始就确定好UI需要展示的信息 
###（三）script脚本结构
- 不过分抽象，没有怎么复用的逻辑不进行抽取

## 生成提醒

- 如果当前目录里还没有 `project.godot`，只有在用户希望得到可直接打开的 Godot 工程时再创建它。
- 量化实例文件夹的名称为对象名，然后文件夹里面有对应的图片和脚本，文件名一致
