# README 模板

按项目实际情况改写，不要逐字照搬。

## 项目名称

一句话概括项目定位，例如：`LuckMonster 是一个使用 Godot 开发的轻量怪物主题游戏原型。`

## 项目简介

- 这个项目要做什么
- 当前已经实现到什么阶段
- 当前更偏向原型验证、核心机制开发还是内容制作

## 当前实现

- 已完成的场景、系统或模块
- 已有的资源目录或 UI 结构
- 如果功能尚未完成，要明确写成“已搭好骨架”或“正在开发中”

## 运行方式

1. 使用 Godot 4 打开项目根目录。
2. 加载 `project.godot`。
3. 运行主场景或默认启动场景。

如果项目还缺少明确的主场景，可以直接说明当前需要在编辑器中手动选择测试场景。

## 目录结构

```text
Core/   核心逻辑
Ex/     扩展功能或实验模块
Res/    资源文件
UI/     界面场景与界面脚本
```

如果实际目录职责不同，要按真实用途重写。

## 开发说明

- 使用的引擎版本
- 提交规范或分支规范
- 哪些目录是手工维护，哪些目录是引擎自动生成

## 后续计划

- 下一步准备实现的核心机制
- 待补的系统、资源或文档
