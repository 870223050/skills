# 游戏 UI 概念图与素材图

## 概述

这个 skill 用来把一份游戏 UI 需求和一组参考图，整理并生成两张配套图片：

- `概念图`：包含标题、按钮文字、说明文案、交互状态等信息的完整 UI 视觉稿
- `素材图`：保留同样的构图、面板、装饰、按钮、插槽和美术风格，但去掉所有可读文字，作为后续切图、叠字或继续细化的纯美术底图

它适合用于前期界面概念设计、外包需求整理、策划案可视化和 UI 实装前的方向确认。

## 适用场景

- 用户提供了游戏界面的功能说明、文案内容和参考图，希望直接生成一张完整视觉稿
- 用户除了要一张带信息的 UI 概念图，还需要一张同款无字素材图
- 需求里包含交互信息，需要在图中体现，例如选中、高亮、禁用、锁定、分页、稀有度、按钮可点击状态等
- 目标是生成光栅概念图，而不是直接编写 Godot、Unity、HTML 或其他引擎中的真实 UI

如果用户真正要的是可运行界面、场景文件、前端页面或引擎组件，而不是概念图，请不要使用这个 skill 作为终点，而应把它作为概念阶段辅助。

## 必须产出

除非用户明确只要其中一种，否则默认同时产出以下两项：

- `概念图`：完整展示界面布局、文字内容和交互状态
- `素材图`：与概念图保持同一套界面结构和风格，但移除所有可读文字，只保留可复用的美术元素

第二张图不是重新设计，而是第一张图的无字纯美术版本。

## 工作流程

### 1. 收集输入信息

开始生成前，优先整理这些信息：

- 界面或场景名称，例如 `relic_reward.tscn`
- 这个界面的用途，以及玩家要做什么操作
- 界面分区和组件结构
- 需要体现出来的交互逻辑
- 需要出现在图中的准确文案
- 美术风格、题材、配色、材质倾向
- 在本地路径：C:\Users\lee2813\Desktop\1231_游戏项目实现暨程序能力提升\luckymosterGd\Res\Picture\Backgrounds 中查看一张或多张参考图
- 额外约束，例如横竖版比例、不要角色立绘、不要现代网页感、文字尽量准确等

如果用户提供的是本地图片路径，先用 `view_image` 把图片加载进上下文，再进行生成。

### 2. 整理需求为生成简报

把用户原始描述整理成简洁但完整的生成简报，至少包含这些部分：

- 界面目标
- 视觉风格
- 布局说明
- 交互状态
- 准确文案
- 避免项

要保留用户原本定义的组件名称和逻辑。不要随意新增面板、功能或玩法，除非只是为了让构图更清晰。

### 3. 生成概念图

优先使用内置 `image_gen`。如果当前会话里也存在 `imagegen` skill，就遵循它的工作流和保存规则。

概念图应满足：

- 画面重点是 UI，而不是背景或角色
- 文字、标题、按钮和说明区尽量可读
- 交互状态明确，例如单选高亮、禁用灰态、锁定描边、当前页签等
- 画面像成熟商业游戏截图，而不是线框图或网页原型
- 如果用户给了准确文案，要尽量按原文生成

如果文案准确性特别重要，要在提示词里明确强调“尽量准确渲染指定文字”。

### 4. 生成素材图

在概念图方向确定后，再根据步骤3生成的图片生成一张同构图的无字素材图，要求：

- 保持同样的镜头、布局、边框、材质、装饰和氛围
- 去掉所有可读文字、数字、按钮字样和说明文案
- 保留空白文本区域、按钮底座、标题牌、卷轴、羊皮纸、插槽、边框和分隔结构
- 保留不依赖文字的交互表达，例如高亮外发光、稀有度颜色、禁用灰态、图标内容、选中描边等

最终结果应该能作为后续叠字、切图、继续绘制或 UI 实装参考的干净底图。

### 5. 保存项目输出

如果这两张图会用于当前项目，不要只停留在默认生成目录里。要把最终选中的结果复制到工作区，并明确告知保存路径。

推荐命名：

- `<scene-name>-concept.png`
- `<scene-name>-art-only.png`

## 质量标准

- 概念图和素材图必须明显属于同一套 UI 设计
- 同一张图里只表达一套清晰的交互状态，除非用户明确要多方案
- 禁用、锁定、选中等状态必须一眼可辨
- 素材图里的文字区域应该是自然留白，不能被乱码或伪文字替代
- 不要无意识生成现代网页后台风，除非用户明确要这种方向
- 图标、边框、材质、颜色、氛围需要保持统一的游戏风格

## 典型触发方式

- “我给你 UI 需求和参考图，你用这个 skill 生成一张带文案和交互信息的概念图，再出一张无字素材图。”
- “帮我把 `shop_popup.tscn` 的界面说明整理成一张游戏 UI 视觉稿，并给我对应的纯美术版本。”
- “根据这张参考图和下面的组件逻辑，做一个卡牌游戏奖励弹窗，既要展示文字信息，也要输出可继续加工的无字底图。”

## 限制

- 这个 skill 不保证文字一定像排版软件那样逐字精确
- 这个 skill 用于概念生成，不负责自动拆出可直接实装的切片素材
- 如果用户后续需要 Godot、Unity 或前端中的真实界面实现，应在概念阶段之后单独推进实现任务
