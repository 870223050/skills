# 游戏 UI 生成提示词模板

## 输入信息清单

有条件时尽量收集这些内容：

- 场景名或文件名
- 界面用途
- 目标平台和画面比例
- 组件列表
- 交互逻辑
- 需要显示的准确文字
- 美术风格关键词
- 每张参考图的作用
- 明确不要出现的内容

## 概念图提示词模板

```text
用途：游戏 UI 概念图
资产类型：全屏游戏界面视觉稿
场景名称：<场景名或文件名>
核心目标：<一句话概括界面目标>
输入图片：<图1：风格参考；图2：布局参考>
视觉风格：<题材、渲染方式、配色、材质语言>
布局说明：<主要区域以及每个区域的内容>
交互状态：<选中、禁用、锁定、hover、分页、稀有度等>
准确文案：<标题、标签、说明文案、按钮文字>
图标与内容说明：<卡牌、遗物、按钮、页签、头像、插槽里应出现什么>
构图目标：<层级、可读性、拥挤度、焦点>
避免项：<不要额外角色、不要水印、不要手机边框、不要错风格等>

生成一张成熟商业游戏风格的 UI 截图式概念图。尽量准确渲染指定文字，并让交互逻辑一眼可辨。
```

## 素材图提示词模板

```text
基于同一套 UI 设计和同一构图，再生成第二张图。
保留面板布局、边框、图标插槽、材质、配色、光照，以及所有不依赖文字的交互提示。
移除所有可读文字、数字、标题、说明文案和按钮字样。
原本放文字的位置要保留自然干净的留白面，不要用乱码或伪文字替代。
不要重新设计界面。这张图应当是第一张概念图对应的无字纯美术底图，供后续 UI 制作使用。
```

## 输出约束

两张图的关系应该是：

- `概念图`：有清晰文字信息和交互状态
- `素材图`：无可读文字，但结构、风格、布局与概念图一致
